# 📋 CLAUDE.md - Документация Проекта

## 🎯 Назначение Проекта

**Свинка-Дружок** - это веб-приложение антистресс-тамагочи со свинкой.

Основная цель: предоставить пользователю спокойный, расслабляющий опыт взаимодействия с виртуальным питомцем.

⚠️ **ВАЖНО**: Этот проект имеет эмоциональную ценность. Любые значительные изменения должны обсуждаться.

## 🏗️ Архитектура

### Модульная структура

```
App (app.js)
├── PetLogic (pet-logic.js)
│   └── SoundManager (sound-manager.js)
└── UIManager (ui-manager.js)
    └── DOM Elements
```

### Модули

#### 1. `js/sound-manager.js`
**Класс**: `SoundManager`

**Отвественность**: Управление звуками с помощью Web Audio API

**Публичные методы**:
- `playTone(frequency, duration, type)` - воспроизвести тон
- `playSequence(frequencies, duration, interval)` - последовательность тонов
- `eatSound()` - звук еды
- `happySound()` - звук счастья
- `playSound()` - звук игры
- `sleepSound()` - звук сна
- `wakeSound()` - звук пробуждения
- `petSound()` - звук ласки
- `sadSound()` - грустный звук
- `toggleSound()` - включить/выключить
- `isEnabled()` - проверить статус

**Особенности**:
- Сохраняет состояние в localStorage
- Автоматически запрашивает разрешение на AudioContext
- Генерирует звуки синтезатором (не использует файлы)
- Безопасен для всех браузеров

#### 2. `js/pet-logic.js`
**Класс**: `PetLogic`

**Ответственность**: Логика состояния и поведения питомца

**Свойства state**:
```javascript
{
  hunger: 0-100,        // Голод (0 = сытая, 100 = голодная)
  happiness: 0-100,     // Счастье
  energy: 0-100,        // Энергия
  isAwake: boolean,     // Проснулась ли
  isSleeping: boolean,  // Спит ли
  lastAction: string,   // Последнее действие
  createdAt: timestamp  // Когда создана свинка
}
```

**Методы действий**:
- `feed()` - кормить (голод -35, счастье +8, энергия -8)
- `pet()` - гладить (счастье +12, энергия -5, голод +3)
- `play()` - играть (счастье +20, энергия -25, голод +12)
- `sleep()` / `wake()` - сон/пробуждение

**Система настроения**:
```
Спящая → sleeping (😴)
Голод > 85 → veryHungry (🤤)
Голод > 65 → hungry (😋)
Энергия < 20 → veryTired (💤)
Энергия < 40 → tired (😪)
Счастье > 85 → veryHappy (🤩)
Счастье > 65 → playful (🎾)
Счастье > 50 → happy (😊)
Иначе → calm (😌)
```

**Автоматические обновления** (каждые 1.5 сек):
```
Если спит:
  - Голод +0.3
  - Счастье +0.2

Если бодрствует:
  - Голод +0.8
  - Энергия -0.6
  - Если голод > 70: счастье -0.8
  - Если энергия < 30: счастье -1.2
```

**Сохранение**:
- Все изменения сохраняются в localStorage
- State восстанавливается при перезагрузке
- `getAge()` возвращает возраст в минутах

#### 3. `js/ui-manager.js`
**Класс**: `UIManager`

**Ответственность**: Управление интерфейсом и визуальными эффектами

**Основные методы**:
- `updateStats(state)` - обновить полосы статистики
- `updateMood(mood)` - обновить настроение
- `updateTime()` - обновить время
- `playAnimation(name)` - запустить анимацию
- `showFeedback(text, duration)` - временное сообщение
- `addHearts(count)` - показать сердечки
- `addSparkles(count)` - показать блёстки
- `addBubbles(text)` - речевой пузырь
- `updateButtonStates(state)` - обновить кнопки
- `setButtonClickHandlers(callbacks)` - привязать обработчики
- `setPetClickHandler(callback)` - клик на свинку

**CSS классы анимаций**:
- `.wiggle` - покачивание (0.4s)
- `.bounce` - прыжок (0.6s)
- `.spin` - вращение (0.8s)
- `.excited` - возбуждение (бесконечное)
- `.eating` - еда (4x встряска)
- `.nod` - кивание
- `.sleeping` - сон (мягкая пульсация)
- `.sad` - грусть (серый фильтр)

**Визуальные эффекты**:
- Сердечки (❤️) с animation float-up
- Блёстки (✨) с animation sparkle-float
- Речевые пузыри с animation bubble-pop

#### 4. `js/app.js`
**Ответственность**: Интеграция и орхестрация

**Основная логика**:
1. `update()` вызывается каждые 400мс
2. Получает состояние от PetLogic
3. Определяет настроение
4. Обновляет UI через UIManager
5. Применяет анимации

**Event Handlers**:
- Кнопки: feed, pet, play, sleep
- Прямой клик на свинку: погладить

## 🎯 Ключевые решения

### Почему нет состояния "болезнь" или "смерть"?
Это антистресс-приложение. Дизайнерское решение приоритизирует позитивный опыт без наказания.

### Почему сохранение в localStorage?
- Позволяет питомцу "существовать" между сеансами
- Создаёт привязанность к свинке
- Простая реализация без бэкенда

### Почему звуки генерируются, а не загружаются?
- Меньше файлов = быстрее загружается
- Нет зависимостей от внешних ресурсов
- Работает везде (даже offline)

### Почему CSS вместо SVG/Canvas?
- Легче редактировать
- Лучше масштабируется
- Меньше кода
- Проще анимировать

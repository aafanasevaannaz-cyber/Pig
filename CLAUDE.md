# Документация Проекта: Свинка-Дружок

## 📋 Назначение

Этот проект - полнофункциональное веб-приложение для антистресс-тамагочи со свинкой.
Это интерактивный питомец, который помогает релаксировать пользователю.

**Важно**: Этот проект имеет культурную и эмоциональную ценность для разработчика.
Любые изменения должны согласовываться с автором.

## 🏗️ Архитектура

### Модули

#### `js/pet-logic.js`
- **Класс**: `PetLogic`
- **Ответственность**: Логика состояния питомца
- **Основные методы**:
  - `feed()` - кормить питомца
  - `pet()` - гладить питомца
  - `play()` - играть
  - `sleep()` / `wake()` - управление сном
  - `autoUpdate()` - автоматическое обновление статистики

**Механика**:
- Голод растёт со временем (даже во сне, но медленнее)
- Счастье зависит от действий пользователя и состояния
- Энергия восстанавливается только во сне

#### `js/ui-manager.js`
- **Класс**: `UIManager`
- **Ответственность**: Управление интерфейсом
- **Основные методы**:
  - `updateStats()` - обновить полоски статистики
  - `updateMood()` - обновить настроение/эмодзи
  - `playAnimation()` - запустить анимацию
  - `setButtonClickHandlers()` - привязать обработчики кнопок

#### `js/app.js`
- **Ответственность**: Интеграция логики и UI
- **Основное**: Функция `update()` вызывается каждые 500мс
- **Цикл**: PetLogic → UI Manager → Обновление экрана

### Стили

#### `styles/main.css`
- CSS переменные для цветов (`:root`)
- Layout с использованием Grid и Flexbox
- Компоненты: кнопки, панели статистики, контейнер питомца
- **Кастомный CSS-питомец**: Вся свинка нарисована CSS (без SVG/изображений)

#### `styles/animations.css`
- Ключевые кадры для анимаций
- Классы для применения анимаций (`.wiggle`, `.bounce`, `.sleeping` и т.д.)
- Анимации сердечек при взаимодействии

## 🎯 Дизайн Решения

### Почему нет состояния питомца, которое ведёт к смерти?

Это **антистресс**-приложение. Нет наказаний, нет чувства вины.
Свинка не умирает, не болеет. Она просто становится грустнее.

### Почему модульная архитектура?

- **Читаемость**: Легко найти нужную логику
- **Тестируемость**: Каждый модуль можно тестировать отдельно
- **Масштабируемость**: Легко добавить новые действия или статистику

### Цветовая палитра

```

Розовый (акцент):     #FF88CC  - яркий, привлекающий внимание

```

## 🔄 Как Работает Цикл Обновления

1. `setInterval` каждые 500мс вызывает `update()`
2. `update()` получает текущее состояние питомца
3. `update()` определяет текущее настроение (mood)
4. UI Manager отображает всю информацию
5. Анимации применяются в зависимости от настроения

**Внутри**: `PetLogic.autoUpdate()` вызывается каждые 2 секунды:
- Голод растёт
- Энергия падает
- Счастье может упасть, если питомец голоден или усталый

## ✅ Принципы Разработки

1. **DRY** - не повторяй код
2. **KISS** - делай просто
3. **No External Dependencies** - только ванильный JavaScript
4. **Soft by Design** - никаких жёстких наказаний
5. **Responsive** - работает везде

## 🐛 Известные Особенности (не баги)

- Свинка не может умереть (это фича!)
- Во время сна питомец не может взаимодействовать
- Сложнее всего сделать свинку счастливой, если она очень голодна

## 📦 Зависимости

Нет внешних зависимостей! Всё работает в браузере.

**Требования**:
- Браузер с поддержкой ES6+
- Поддержка CSS Grid и Flexbox
- Поддержка CSS переменных

# Звуки

## Принцип

Звук у свинки не должен быть громким, резким, аркадным или тревожным.
Это антистресс‑дружок, поэтому звук нужен как мягкий отклик:

- короткий
- тёплый
- нечастый
- без крика и перегруза
- приятный даже в повторе

## Что нужно сделать

### 1. Базовые звуки

- `purr-soft` — мягкое урчание на поглаживание
- `wheek-happy` — короткий довольный писк на радость
- `munch-soft` — тихое хрумканье на еду
- `sleepy-breath` — спокойный сонный выдох
- `sniff-soft` — маленький носовой шорох
- `house-rustle` — тихое шуршание домика
- `ui-soft-tap` — очень деликатный отклик кнопки

### 2. Где какой звук нужен

#### Просто рядом

- почти тишина
- иногда очень тихий `sniff-soft`
- редкое едва заметное шуршание

#### Гладить

- основной звук: `purr-soft`
- можно добавить очень тихий второй слой телесного шороха
- не включать длинный бесконечный луп
- лучше короткие мягкие фразы звука по касанию

#### Покормить

- короткий радостный `wheek-happy`
- затем `munch-soft`
- при серии касаний лучше маленькие вариации, чтобы звук не бесил

#### Домик

- `house-rustle`
- при уходе внутрь — тихий шорох
- потом почти тишина

#### Сон

- `sleepy-breath`
- очень редкий и негромкий
- без мелодии, без колокольчиков, без сказочной липкости

#### Милая радость

- короткий `wheek-happy`
- можно второй маленький отклик выше тоном
- без визга и без мультяшного удара

## Что нельзя делать

- громкие писки
- резкие щелчки
- постоянный фон без остановки
- звук каждый раз на любое движение
- длинный раздражающий луп
- тревожные или жалобные звуки как будто свинка страдает
- слишком реалистичные громкие животные звуки

## Технический подход

Сейчас для прототипов допустимы два варианта:

### Вариант A — Web Audio API

Подходит для быстрых мягких синтетических откликов:

- урчание
- писк
- тихий выдох
- шорох

Плюсы:

- легко менять громкость и тон
- не нужны внешние файлы для первых тестов
- удобно делать короткие отклики

Минусы:

- легко получить искусственный неприятный звук
- для финала часть звуков всё равно лучше записать или подготовить отдельно

### Вариант B — отдельные аудиофайлы

Финально лучше держать звуки файлами в проекте.

Папка:

`app/assets/sounds/`

Нужные файлы:

- `purr-soft.mp3`
- `wheek-happy.mp3`
- `munch-soft.mp3`
- `sleepy-breath.mp3`
- `sniff-soft.mp3`
- `house-rustle.mp3`
- `ui-soft-tap.mp3`

## Правила громкости

Все звуки должны быть тише обычного мобильного интерфейса.

Ориентир:

- базовые реакции: очень тихо
- радость: чуть ярче, но коротко
- сон и домик: почти у порога слышимости
- никакого клиппинга и резкой атаки

## Поведение по частоте

- не повторять один и тот же звук слишком часто
- ввести маленький cooldown
- для поглаживания не запускать новый звук каждые 10 мс
- часть звуков делать случайно редкими, а не обязательными

## Итог

Звуковой характер свинки:

- спокойный
- тёплый
- телесный
- нежный
- не мультяшно-истеричный
- без ощущения долга и тревоги
